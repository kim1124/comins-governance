# Comins Brand

## Definition

Comins is a brand for independently released npm frontend modules that solve focused interface and interaction problems for React applications.

Each module must be useful without a Comins runtime, a shared design system, or another Comins module. The product promise is clear public APIs, predictable integration, accessible interaction, and production-focused verification.

## Development Transparency

Comins modules may be developed with Codex AI assistance under maintainer review. AI assistance is an engineering practice, not a runtime dependency or a substitute for release ownership.

## Naming

- Use `Comins <Module Name>` in public documentation and playgrounds.
- Prefer unscoped npm package names in the `comins-<domain>` form when available.
- Prefer GitHub repository names that align with the public package name. Preserve established repository names unless an explicit migration is approved.
- Do not use KMSF as a current product identity in public Comins documentation.

## Visual Identity

- Module runtime UI must not inject a Comins logo, favicon, font, or marketing copy into consuming applications.
- Shared icons and a public wordmark are deferred until at least three public modules and a shared documentation surface exist.
- Documentation and playground branding must describe the module being used, not require consumers to adopt a Comins visual system.
